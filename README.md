# Birthday Voting Game

משחק יום הולדת אינטרנטי עם:
- מסך מארח
- כניסה מהנייד דרך QR או קישור
- סרטון לכל סבב
- פתיחת הצבעה אחרי הסרטון
- תוצאות חיות
- שמירה של סשנים, הצבעות וסרטונים על השרת

## איך מכניסים סרטונים

יש 2 דרכים:

1. העלאה מתוך המסך
פשוט לוחצים על `העלה סרטון לסבב` במסך המארח.
הקובץ נשלח לשרת ונשמר בתיקיית האחסון.

2. קבצים קבועים מתוך `public/videos`
שמים קובץ בתיקייה `public/videos`, ואז מזינים בשדה הווידאו נתיב כמו:

```text
/videos/round-1.mp4
```

## פיתוח מקומי

התקנה:

```bash
npm install
```

פיתוח Frontend בלבד:

```bash
npm run dev
```

הרצת השרת שמגיש גם את ה־API וגם את ה־build:

```bash
npm run build
npm run start
```

ברירת המחדל המקומית:
- אפליקציה: `http://localhost:4173`

## אחסון

השרת שומר מידע בתיקיית האחסון:
- סשנים והצבעות: `sessions.json`
- סרטונים שהועלו: `uploads/`

ברירת מחדל מקומית:

```text
storage/
```

אפשר לשנות בעזרת משתנה סביבה:

```text
STORAGE_DIR
```

## פריסה קבועה ב-Railway

הפרויקט מוכן ל־Railway דרך `Dockerfile` ו־`railway.json`.

### מה חשוב ב-Railway

צריך ליצור:
1. Service אחד מה־repo הזה
2. Volume קבוע שממופה ל־`/data`

### משתני סביבה מומלצים

אין חובה להגדיר `PORT`, Railway מגדיר אותו לבד.

כדאי להגדיר:

```text
STORAGE_DIR=/data
```

### תהליך העלאה

1. מעלים את הפרויקט ל־GitHub
2. ב־Railway בוחרים `New Project`
3. בוחרים `Deploy from GitHub repo`
4. מחברים את הריפו הזה
5. Railway יזהה את `Dockerfile`
6. מוסיפים Volume וממפים אותו ל־`/data`
7. מגדירים Environment Variable:

```text
STORAGE_DIR=/data
```

8. עושים Deploy

### בדיקת תקינות

אחרי העלייה, הכתובת הזאת צריכה לעבוד:

```text
/health
```

## מבנה חשוב בפרויקט

- `src/App.tsx` - ממשק המארח והניידים
- `src/App.css` - עיצוב האפליקציה
- `server.mjs` - שרת האפליקציה וה־API
- `public/videos/` - סרטונים קבועים מראש
- `storage/` - נתונים וסרטונים מקומיים

## למה זה הפתרון הנכון לאירוע

כי הוא לא תלוי במחשב שלך:
- האתר רץ על שרת קבוע
- הסרטונים נשמרים על volume קבוע
- ההצבעות נשמרות על השרת
- המשתתפים יכולים להיכנס מכל מכשיר
